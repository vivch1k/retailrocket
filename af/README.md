# Учебный антифрод-проект

В папке находятся:

- `raw_login_events.csv` — сырые события логина;
- `antifraud_starter.ipynb` — задания с TODO;
- `antifraud_solution.ipynb` — полностью выполненное и проверенное решение;
- `requirements.txt` — минимальные зависимости.

## Сценарий

Одна строка — одна попытка логина. Модель строится для оценки риска сразу
после попытки аутентификации. Таргет `is_fraud` подтверждается позже.

Запрещено использовать как признаки:

- `is_fraud`;
- `fraud_confirmed_at`;
- данные из будущих событий;
- итоговые агрегаты по всему пользователю без временного ограничения.

## Как запустить

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
jupyter lab
```

Сначала открой `antifraud_starter.ipynb`. Решение используй для сверки.

## Что должно получиться

В эталонном решении логистическая регрессия дает PR-AUC примерно 0.65–0.75
на будущих данных. Точное значение может немного отличаться между версиями
библиотек.

Главная цель проекта — не максимальная метрика, а правильная временная
логика признаков и валидации.
